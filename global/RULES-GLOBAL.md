# Financial Rules — Global

These rules apply to ALL workspaces. They cannot be overridden.

## Data integrity

1. **No fabrication** — never invent financial data, projections, or metrics
2. **Source required** — every number needs a source (bank statement, invoice, API)
3. **Assumptions explicit** — every forecast states its assumptions clearly
4. **Reconciliation** — bank balance must match book balance before reporting
5. **Audit trail** — every change to financial data is logged with date and reason

## Compliance

1. **Not tax advice** — FINANCE:OS helps organize data, not give tax advice. Flag for CPA.
2. **Not legal advice** — flag contract and regulatory questions for counsel
3. **GAAP awareness** — follow generally accepted accounting principles where applicable
4. **Jurisdiction matters** — tax rules vary by location. Always confirm jurisdiction.
5. **Data privacy** — never expose bank account numbers, SSNs, or sensitive financial data in logs

## Reporting standards

1. **Consistent periods** — use consistent date ranges for comparisons
2. **Accrual vs cash** — clearly state which basis is being used
3. **Rounding** — round to nearest dollar for reports, keep cents in calculations
4. **Negative numbers** — use parentheses for losses: ($1,234) not -$1,234
5. **Currency** — always specify currency for international workspaces

## Budget discipline

1. **Budget exists before spending analysis** — create the budget first
2. **Variance is expected** — report it honestly, explain it clearly
3. **Reforecast quarterly** — budgets are living documents
4. **Zero-based optional** — don't assume last year's budget is correct

## Measurement

1. **Track against plan** — measure actuals against budget
2. **Trend over snapshot** — month-over-month matters more than single months
3. **Unit economics** — track per-unit metrics, not just totals
4. **Report honestly** — if numbers are bad, say so. Learnings come from honesty.
